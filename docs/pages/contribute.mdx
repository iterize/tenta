## Development setup

- Clone the repository and switch into the `server` directory
- Install the Python version noted in `.python-version` (e.g. with `pyenv`)
- Install the [poetry package manager](https://github.com/python-poetry/poetry)
- Install the dependencies with `./scripts/setup`
- Make sure that you have Docker installed
- You can now start the server locally with `./scripts/develop`; This automatically spins up local PostgreSQL and Mosquitto instances with Docker; The server will be available at `http://localhost:8421`
- You can run the tests with `./scripts/test` and format and lint the code with `./scripts/check`

## Dashboard

- Clone the repository and switch into the `dashboard` directory
- Have [NodeJS](https://nodejs.org/en) version >= 16 installed on your machine
- Install the dependencies with `npm install`
- Use `.env.template` to create a `.env.local` file on the same level
- Run the development server with `npm run dev`
- Create a production build with `npm run build`
- The directory structure is explained by [NextJS](https://nextjs.org/) (version 13, app router)

## Documentation

- Clone the repository and switch into the `dashboard` directory
- Have [NodeJS](https://nodejs.org/en) version >= 16 installed on your machine
- Install the dependencies with `npm install`
- Run the development server with `npm run dev`
- The docs are build using [Nextra](https://nextra.site/) which is again based on [NextJS](https://nextjs.org/)

## Making a pull request

You can create a draft pull request if your your contribution is not yet ready to merge. Please check if your changes call for updates to the documentation! Changes to the server should be accompanied by tests.
